Offer of services

Introduction

Eno is a tool that generates survey questionnaires starting from their formal description in DDI. It automates the production of “collection instruments” (paper or web forms, collection modules for surveyors, etc.) based on a formal and abstract description of the questionnaire as standardised metadata. These models are then used to produce the collection media, in different modes, within production collection infrastructures. Eno is used by Insee since 2015 : more than 30 business surveys are using Eno for their web and paper questionnaires, and the offer for household surveys is growing.

A metadata driven tool

The use of standard description in DDI provides and enforces structured information (and consequently leads to well structured questionnaire). This choice of DDI as a modelling format for questionnaire specification was motivated by : - A strong need for a precise modelisation of the questionnaire in order to be able to render its content and logical flow in different collection modes (i.e a semantically rich model) - DDI being an open standard - DDI being serialised in XML, and thus easily processable.

Since 2015, Eno takes as input the formal description of the questionnaire complying with the DDI standard and executes a completely automated chain of transformations to produce the current survey questionnaire (web and paper questionnaires).

Eno produces survey questionnaires

DDI description enables to identify clearly the different components of a questionnaire, such as sequences of questions, questions, instructions, different formats of response.

Common used response formats are available : simple text, number, date, single choice question, multiple choice question, grid question.

It is possible to incorporate dynamic layout : filters and questionnaire logic flow, loops, controls to check the format of the input fields, consistency between the different input fields and, where necessary, with previous collected data. Controls are emphasized by probing messages to invite respondent to correct or fulfill his response.

Outputs of Eno : the questionnaires can be created in different formats from the same description : XForms web questionnaires (executed on Orbeon Forms Runner), fo output format for paper questionnaires (the FO is transformed into PDF by a module based on Apache FOP), or questionnaires in a dedicated format which will be used for face-to-face or telephone collection.

When they are personalized, the "collection instruments" can be pre-filled with previously collected data or from a register by instance, and labels can be customized with informations on the respondent, already known before data collection as well as collected previously in the form.

Objectives: standardization, industrialization, reuse

Eno is a generator and meets the need of standardization and industrialization

The development of one tool, based on standardised description in DDI leads to a standardized design. It ensures that questionnaires comply with standards and recommendations on survey design. The distinction between questionnaire's elements allows to show them in different ways and styles, and to create a coherent layout for questionnaires produced.

Display choices are also shared between surveys within the NSI, improvements are available/profitable for every users at the same time. For the household or business surveyed,it enables them to better identify surveys that are indeed within the official statistics remit.

The generation of different data collection formats from the same questionnaire description supports unimode design for mixed mode surveys. The description is done once : the use of the generator lightens the burden of work for the survey departments.

Reuse

The questionnaire documentation can be stored and reused, in a global NSI context, or for a similar survey from year to year. Classifications are shared between different surveys teams.

Eno is a stakeholder in active metadata : as the end of the "survey process", the questionnaire description can also (and will) be reused for dissemination or documentation of surveys.

Roadmap

Eno is very widely, even systematically, used in business surveys (web questionnaire, paper questionnaire, takeover and management tool). Household surveys have also started to use it and work is underway to increase the functional coverage required for these surveys (e.g. description of filters, possibility to loop or select occurrences of a loop, interviewer questionnaire).

How to get it?

Eno is published as open source on a GitHub repository. It is available under the MIT license, and contributions as well as suggestions for improvements are welcome.

It is developped in an international perspective, codes and comments are in english. Data of course can be collected in any langage.

How to use it?

Eno comes packaged in various ways: as a command-line tool, a REST web service and a Docker image.

ENO is often associated with Pogues. This is is an additional module to Eno, a questionnaire design user interface, that connects with the generation process. Pogues produces the DDI description of the questionnaire : using this tool, a questionnaire designer can specify his web questionnaire in a friendly way and visualize the generated result in one click.
